# Council protocol

## 0. Exclusion precedence

Before checking either bridge or creating a run, classify external-human rulings
as `human-ruling-required`, missing authority as `requires-new-authorization`, and
an active calculation as monitoring rather than a blocker. Do not convene a
council for those states. This exclusion overrides explicit invocation wording.

## 1. Freeze the blocker packet

Select one to five blockers from current authoritative artifacts. Record the exact
terminal, evidence paths and hashes, completed attempts, legal machine actions,
forbidden shortcuts, and a falsification criterion. Give both teams byte-identical
packets. Do not include the other team's ideas.

## 2. Independent proposal round

Run three roles per model family when resources allow:

1. theoretical/geometry physicist;
2. lattice, statistics, and numerical-methods physicist;
3. adversarial convention and falsification reviewer.

Require one schema-valid memo per role and blocker. Cap parallel Fable sessions at
two unless the user approves a higher account load. Native subagents may run within
the host's concurrency limit. Return summaries, not raw reasoning traces.

## 3. Blind cross-review

Strip provider, model, team, session, and author labels. Send each accepted proposal
to an opposite-family reviewer. Require the reviewer to identify the weakest
inference, hidden imports, convention dependencies, counterexamples, whether the
test distinguishes alternatives, and a stronger repair if one exists.

## 4. Rebuttal

Return the critique to the original participant. Permit revision, claim narrowing,
conversion to a negative test, or withdrawal. Never erase the original memo.
Allow at most two critique/rebuttal rounds.

## 5. Adjudication

Score each surviving proposal from 0 to 5 on exact evidence support, mathematical
consistency, falsifiability, expected information gain, computational feasibility,
and convention robustness. Record gate compliance separately as pass/fail.
Consensus is evidence, not authority. Prefer the cheapest test that decisively
separates live alternatives and preserves every firewall.

## 6. Exit

Return a ranked recommendation and fallback, unresolved contradictions, cost and
storage bounds, required authorization, and the evidence that would change the
classification. Mark the council ineffective if it merely restates the blocker.
Do not automatically invoke another council on the same evidence state.
