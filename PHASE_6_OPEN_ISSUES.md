## Scientific Limitations

### P6-001 Quantitative Success Is Still Toy-Placeholder Success

Evidence:

- `consistency_scorecard.json` reports `5/5` passing matches,
- the report explicitly states all targets are synthetic toy placeholders.

Impact:

- the clean quantitative result is useful as a control study,
- it is not evidence against or for physical observables.

### P6-002 The Checked-In Reference Campaign Is Still Toy-Environment Only

**CLOSED by P6-M4.** The reference campaign now includes toy, structured 4x4,
and imported (synthetic example) environment records. `environmentSummary`
in the dossier shows 3 entries.

### P6-003 The Primary Reference Campaign Is Not Yet Background-Bridged

Evidence:

- the successful campaign consumed pre-authored branch/refinement value tables,
- the artifact set does not show Phase III background-derived branch inputs as
  the authoritative source.

Impact:

- the reference evidence still depends on curated tables rather than the
  upstream persisted background atlas.

### P6-004 Identity-Only Shiab Still Dominates The Evidence Path

Evidence:

- the campaign provenance and branch IDs in the successful run remain
  `identity-shiab` variants,
- no non-identity Shiab variation appears in the checked-in reference study.

Impact:

- convergence and robustness evidence remain local to the identity-only Shiab
  slice.

## Infrastructure Limitations

### P6-005 The Standard Campaign Spec Still Undershoots The Intended Contract

**CLOSED by P6-M1/P6-M3.** `campaign.json` now uses `schemaVersion = "1.1.0"`
and declares all four sidecar paths. Sidecar files are generated by
`build-phase5-sidecars` and committed to config/.

### P6-006 There Is No First-Class Campaign Validator

**CLOSED by P6-M1.** `gu validate-phase5-campaign-spec --spec <f>
[--require-reference-sidecars]` is now implemented. `run-phase5-campaign`
accepts `--validate-first` to run it as a preflight gate.

## Evidence And Provenance Limitations

### P6-007 Observation-Chain Evaluation Is Not Present In The Successful Dossier

**CLOSED by P6-M3/P6-M5.** `observationChainSummary` is now present in
`phase5_validation_dossier.json`. Sidecar channels are status="skipped" (empty
arrays), which is explicitly distinguishable from absent.

### P6-008 Falsifier Outputs Do Not Show Coverage, Only Trigger Counts

**CLOSED by P6-M3/P6-M5.** `falsifier_summary.json` now carries
`observationRecordCount`, `environmentRecordCount`, `representationRecordCount`,
and `couplingRecordCount` from the full `FalsifierSummary`. All are 0 because
sidecar channels are skipped, but the distinction is explicit.

### P6-009 Reproduction Metadata Still Uses Placeholder Command Arguments

**CLOSED by P6-M5.** `run-phase5-campaign` now copies all input files to
`<out-dir>/inputs/` and writes a concrete reproduction command referencing
`inputs/campaign.json` into `dossiers/study_manifest.json`.

## Intentionally Deferred Or Not Yet Exercised

### P6-010 Imported-Tier Evidence Exists In Code But Not In The Standard Campaign

The environment provenance fields are implemented, but the successful reference
campaign does not yet consume imported data.

### P6-011 Non-Gaussian Scoring Exists In Code But Not In The Standard Campaign

The target-matching code supports non-Gaussian models, but the successful
reference campaign remains Gaussian and toy-placeholder.

### P6-012 Deferred Falsifier Channels Exist In Code But Not In The Standard Campaign

Observation, environment variance, representation content, and coupling
inconsistency falsifiers are implemented and tested, but the standard campaign
does not yet provide their inputs.

## New Open Issues From Phase VI Implementation

### P6-013 SidecarGenerator Does Not Populate Records From Inputs

The `build-phase5-sidecars` command generates empty-but-present sidecar files
(status="skipped") rather than deriving actual records from the registry,
observables, and environment inputs passed to it.

**Next step:** Implement record derivation logic in `SidecarGenerator` — route
registry candidates to `ObservationChainRecord` entries, environment records to
`EnvironmentVarianceRecord` entries, etc.

### P6-014 FalsifierSummary.EvaluationCoverage Not Threaded Through Campaign Runner

`FalsifierSummary.EvaluationCoverage` is always null in artifacts because
`Phase5CampaignRunner` does not receive `SidecarSummary` from the CLI layer.

**Next step:** Add an optional `SidecarSummary? sidecarSummary` parameter to
`Phase5CampaignRunner.RunFull` and wire it into `FalsifierEvaluator`.

### P6-015 Imported Environment Is Synthetic Example Only

`env_imported_example.json` uses `datasetId=synthetic-example-dataset-su2-2d-001`
and `sourceHash=sha256-0000...`. This satisfies schema checks but does not
represent real imported geometry.

**Next step:** Replace with genuine provenance fields when a real external mesh
dataset becomes available.
