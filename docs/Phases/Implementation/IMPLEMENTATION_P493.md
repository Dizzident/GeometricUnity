# Implementation P493 — Phase456 Stored-Artifact Failure Decomposition

Phase493 is a deterministic, read-only forensic consumer of the immutable
Phase456 production record. It byte-binds the raw and summary production
twins, Phase456 output, consumer and analyzer source, explicit authorization,
and concatenated preregistration pack. It launches no sampler or benchmark and
does not modify or reinterpret the Phase456 terminal.

The phase independently reapplies the frozen `T=4` cosh-estimator domain to
the retained full correlators, records effective sample sizes against the
committed floor, and reconstructs the any-invalid-row family-firewall
propagation. Primitive energy/virial gates and the exact-Gaussian pipeline are
kept distinct from the observable-level free-field failure: passing primitive
checks does not prove sampler correctness, while an undefined mass comparison
does not by itself support a sampler implementation defect.

The exact terminal taxonomy is `sampler-defect-supported`,
`estimator-domain-failure-supported`, `statistics-limited-supported`,
`mixed-failure-supported`, `failure-source-unresolved`, or
`invalid-committed-artifact`. Multiple independently supported components use
the mixed terminal. Channel sign/domain evidence is recorded without declaring
the underlying observable physically invalid.

This is A8 exploration evidence only. It cannot rehabilitate Phase456, satisfy
Phase458, construct Phase481, authorize sampling or production, fill a source
contract, or support a physical-unit claim.
